Table 表格 用于展示多条结构类似的数据，可对数据进行排序、筛选、对比或其他自定义操作。

  • 操作栏内的操作不要使用图标按钮，统一使用文字按钮。并且能简短并清楚表达该操作，不能超过6个字。.ql-button.ql-button--text.ql-button--small
  • 列表一列数不可控，默认展示范围：3-8列，若出现更多，可固定重要列，剩余列滚动条展示
  • 列表的宽度占比宽度自适应，但根据字段的重要性显示，重要字段优先完整显示
  • 表头列标题最多输入8个字符
  • 文字类信息根据产品要求字数，超过宽度限以”…“代替，鼠标经过展示全部内容
  • 数字/英文/符号的组合形式，表示编号类信息的字符全部展示，不可省略
  • 表格某部分无数据时用”-“来填充显示，不能出现”null””undefined”，对于空字符串””数据显示为空白。对于数据为零的单元格，填上0
  • 前端数据的重组和数值替换，应当在nodeJs端完成，不得在angualr端转换和重组。
  • 新建和批量操作的按钮组在表格上显示，并右对齐。
  • 复杂查询条件的组在新建和批量操作按钮组之上显示，并左对齐。
  • 简单查询条件的组与新建和批量操作按钮组在同一个row，各占50%显示。
  • 开发人员使用时请统一使用 [stripe]="true" [border]="true" 带边框和斑马纹的表格样式
  • 表格内容超过一屏需要显示竖向滚动条时，需要固定表头和页码。只需滚动表体内容部分

基础用法

如果存在 ql-table-column 未设置宽度，将会自动计算宽度，同时也不会固定显示表格的总宽度。
当数据为 HTML 时，在<ql-table-column>中使用[render-html]="true"
